# 分子对接可复用工作流设计

## 目标

建立一个面向 Linux 的模块化分子对接仓库。用户在新项目中只需修改靶点表、配体目录和配置文件，即可复用完整流程，无需重新拼接命令。流程必须同时支持“按 PDB ID 自动下载结构”和“使用用户提供的 PDB/mmCIF”两类常见入口，也不能假定受体一定带有共晶配体。Windows 仅支持可可靠跨平台的辅助步骤，不承诺 AutoDockTools/PyMOL 命令行主流程。

## 支持范围

主流程包括：结构获取或导入、受体结构检查与清理、可用时保留原配体、受体 PDBQT 制备、配体 PDBQT 制备、多策略口袋定义、AutoDock Vina 批量对接、与当前口袋证据相匹配的质量控制、PLIP 批量相互作用分析、结果汇总。

不纳入主流程：依赖人工审美判断的 PyMOL 成图排版、原始会话记录、受许可限制的软件安装包、大体积结构结果。PyMOL 检查仅提供可选辅助命令和检查清单。

## 用户接口

仓库提供一个主入口和若干独立步骤。用户复制示例配置后，只需维护以下输入：

- `config/config.yaml`：软件路径、Vina 参数、盒子大小、筛选阈值和并发设置。
- `config/targets.tsv`：靶点名、结构来源、PDB ID 或本地结构路径、保留链、可选共晶配体、口袋策略等。
- `inputs/ligands/`：待对接小分子，优先支持 MOL2/SDF。

主入口支持环境检查、全流程运行、指定步骤运行和从已有结果继续。所有生成文件进入 `work/` 或 `results/`，输入文件不被覆盖。

## 组件边界

1. 环境检查器验证 PyMOL、Open Babel、MGLTools、Vina、Python 和 PLIP 是否可调用，并输出可操作的缺失项提示。
2. 结构准备器按配置下载 PDB 或复制用户提供的 PDB/mmCIF，保存原始结构、标准化结构、清理后 PDB、可用的参考配体和受体 PDBQT；每一步都有清单记录，原文件不被覆盖。
3. 配体准备器统一分子格式、生成三维构象并输出 PDBQT，同时保留名称映射。
4. 口袋生成器支持多种互斥策略，并把策略与证据等级写入清单：
   - `co_crystal`：从当前结构的共晶配体计算中心，优先级最高。
   - `explicit_box`：使用用户提供的中心和尺寸，适合已有文献或实验位点信息的情况。
   - `reference_ligand`：把同源或参考结构的配体/口袋映射到目标结构，必须输出比对与映射检查结果。
   - `residue_box`：根据用户提供的关键残基集合计算中心和包围盒。
   - `predicted_pocket`：导入口袋预测工具结果，由用户选择候选口袋；工作流记录工具、排名和分数。
   - `blind`：在无位点信息时覆盖整个受体或指定大区域，仅作为探索性方案，结果不得与共晶口袋对接同等级解释。
   自动模式按上述顺序选择第一个信息充分的策略；若存在多个预测口袋，不擅自替用户挑选，生成候选表并要求明确选择。
5. 对接调度器按靶点与配体笛卡尔积运行 Vina，跳过已经完整生成的结果，失败项单独记录。
6. 质量控制器解析结合能，并按口袋策略采用不同检查：有参考配体时计算构象中心距离；显式坐标或残基口袋时检查构象是否落入盒内及关键残基邻近性；预测/盲对接时检查边界效应、聚类与口袋一致性。输出高可信、探索性、需人工检查和失败四类结果，禁止在没有原配体时伪造“到原配体距离”。
7. PLIP 调度器仅处理筛选后的复合物并生成标准化汇总。

## 数据流与目录

原始输入只读进入 `inputs/`；中间结构、日志和任务状态进入 `work/`；最终表格、筛选结果和报告进入 `results/`。每个阶段生成机器可读清单，后续阶段只消费上一步清单，避免依赖模糊的文件名匹配。

## 错误处理与断点续跑

每个任务采用明确状态：`pending`、`running`、`success`、`failed`、`skipped`。单个靶点或配体失败不会静默丢失；批量流程继续运行并在结尾返回失败汇总。已有文件只有在通过基本完整性检查后才会跳过。用户可用 `--force` 重算指定步骤。

## 验证

在没有商业或外部软件的环境中，验证配置解析、TSV 校验、本地/下载结构入口、PDB/mmCIF/PDBQT 坐标解析、各口袋策略、盒子中心计算、Vina 日志解析、条件化质量控制和筛选规则。测试必须覆盖有共晶配体、无共晶配体但有显式坐标、只有关键残基、只有预测口袋以及盲对接五种情形。Shell 脚本执行语法检查；外部程序调用用小型模拟程序验证参数与断点逻辑。README 明确区分“自动测试通过”和“需要用户本机软件验证”。

## 发布边界

仓库不包含个人绝对路径、原始 JSON、私有链接、下载的第三方结构全集或软件二进制。默认配置使用占位路径与公开示例，`.gitignore` 排除工作目录、结果大文件、许可证文件和本机配置。
